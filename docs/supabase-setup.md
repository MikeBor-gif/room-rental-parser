# Настройка Supabase (база данных бота)

Supabase — бесплатный хостинг Postgres с веб-админкой. Здесь живут пользователи
бота, их фильтры, объявления, очередь доставки и платежи. Настройка занимает
~5 минут и делается один раз.

## Шаг 1. Создать проект

1. Зайти на [supabase.com](https://supabase.com) → **Start your project** →
   войти через GitHub.
2. **New project**:
   - **Name**: например `rental-bot`
   - **Database Password**: сгенерировать и сохранить (понадобится только для
     прямого доступа к Postgres, бот её не использует)
   - **Region**: `Central EU (Frankfurt)` — ближайший к Беларуси
3. Дождаться создания проекта (~2 минуты).

## Шаг 2. Создать таблицы

1. В левом меню открыть **SQL Editor** → **New query**.
2. Скопировать содержимое файла [`deploy/supabase_schema.sql`](../deploy/supabase_schema.sql)
   целиком, вставить и нажать **Run**.
3. Должно появиться `Success. No rows returned`.
4. Проверить: в левом меню **Table Editor** — должны быть таблицы
   `users`, `filters`, `listings`, `deliveries`, `payments`, `feedback`,
   `bot_state`.

Скрипт идемпотентен — его можно безопасно выполнять повторно (например, после
добавления новых таблиц в будущих версиях).

## Шаг 3. Взять ключи доступа

1. В левом меню: **Project Settings** (шестерёнка) → **API**.
2. Понадобятся два значения:
   - **Project URL** — вида `https://abcdefgh.supabase.co`
   - **service_role key** (раздел *Project API keys*, кнопка *Reveal*) —
     длинный ключ, начинается с `eyJ...`

⚠️ **service_role key даёт полный доступ к данным.** Никогда не коммитить его
в репозиторий и не вставлять в код — только в GitHub Secrets (шаг 4) или в
локальный `.env` (который в `.gitignore`).

## Шаг 4. Добавить секреты в GitHub

В репозитории: **Settings → Secrets and variables → Actions → New repository secret**:

| Secret | Значение |
|---|---|
| `SUPABASE_URL` | Project URL из шага 3 |
| `SUPABASE_SERVICE_KEY` | service_role key из шага 3 |

## Шаг 5. Локальный запуск (для разработки)

В файл `.env` в корне проекта (создать из `.env.example`, если его нет):

```
SUPABASE_URL=https://abcdefgh.supabase.co
SUPABASE_SERVICE_KEY=eyJ...
```

## Админка

**Table Editor** в Supabase — готовая админ-панель:

- `users` — кто зарегистрирован, тариф, до какой даты оплачено;
- `feedback` — отзывы пользователей (кнопка «💬 Отзыв» / `/feedback`);
- `payments` — все платежи и их статусы (премиум пока выключен);
- `filters` — какие фильтры настроили пользователи;
- `deliveries` — что и кому отправлено.

Список пользователей можно быстро посмотреть и из чата: админская команда
`/users` присылает последние регистрации.

Любое поле можно править прямо в браузере (например, руками продлить
`paid_until` пользователю).

## Лимиты бесплатного плана Supabase

- 500 МБ БД — при автоматической очистке старых объявлений (встроена в
  scrape-цикл) хватает на годы работы;
- проект «засыпает» после 7 дней без запросов — наш cron ходит в БД каждые
  ~5 минут, так что это не случится.
