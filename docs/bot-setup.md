# Оформление бота в @BotFather

Почти вся «витрина» бота настраивается **автоматически** самим ботом через
Bot API при первом прогоне workflow `bot`. Вручную в
[@BotFather](https://t.me/BotFather) остаётся только имя и аватар.

## 1. Описание и короткое описание — настраиваются АВТОМАТИЧЕСКИ

Ничего делать не нужно: бот сам проставляет описание (видно до нажатия
«Старт») и короткое описание профиля через `setMyDescription` /
`setMyShortDescription`. Тексты заданы в `src/bot/texts.py`
(`BOT_DESCRIPTION`, `BOT_SHORT_DESCRIPTION`). Чтобы изменить — отредактируйте
их и увеличьте `BOT_PROFILE_VERSION`: бот перепроставит на следующем прогоне.

## 2. Кнопка «Menu» (☰) у поля ввода — настраивается АВТОМАТИЧЕСКИ

Ничего делать не нужно: при первом прогоне workflow `bot` бот сам
регистрирует список команд (`setMyCommands` + `setChatMenuButton`), и
Telegram показывает кнопку «☰ Menu» рядом с полем ввода у всех
пользователей.

Список команд задан в `src/bot/texts.py` (`BOT_COMMANDS`). Чтобы изменить —
отредактируйте список и увеличьте `BOT_COMMANDS_VERSION`: бот перерегистрирует
команды на следующем прогоне.

(Админ-команды /approve, /reject, /grant, /stats в меню не выносим —
они работают только для ADMIN_CHAT_ID.)

## 3. Имя и аватар

Имя: `/setname` → выбрать бота → отправить имя.
Аватар: `/setuserpic` → выбрать бота → отправить квадратную картинку
(минимум 512×512). Идея: дом/ключи на однотонном фоне — хорошо читается
в маленьком кружке чата.

## 4. Проверка

1. Откройте бота «глазами пользователя»: описание видно до старта,
   команды подсказываются при вводе «/».
2. `/start` → пройти конструктор фильтра → убедиться, что фильтр появился
   в /filters и в таблице `filters` в Supabase.
3. `/feedback` → отправить текст → в чат админа (`ADMIN_CHAT_ID`) пришёл
   отзыв, а строка появилась в таблице `feedback` в Supabase.
4. `/users` от админа → пришёл список зарегистрированных пользователей.
