# Нативная кнопка «Menu» (☰) у поля ввода — автонастройка через Bot API

**Дата:** 2026-07-15 11:45
**Файлы:** src/telegram.py, src/bot/texts.py, src/jobs/updates.py, tests/test_menu_button.py, docs/bot-setup.md
**Severity:** low

## Problem

У бота не было нативной кнопки «☰ Menu» рядом с полем ввода — пользователь
не видел списка команд без ручной настройки через BotFather, которую легко
забыть выполнить (и она не переживает смену списка команд).

## Root Cause

Регистрация команд была оставлена как ручной шаг в docs/bot-setup.md
(/setcommands в BotFather), хотя Bot API позволяет делать это программно:
setMyCommands регистрирует команды, после чего Telegram сам показывает
кнопку «Menu»; setChatMenuButton({type: commands}) включает её явно.

## Solution

TelegramApi получил set_my_commands() и set_chat_menu_button_commands().
Список команд — BOT_COMMANDS в texts.py с версией BOT_COMMANDS_VERSION.
ensure_menu_button() в updates.py вызывается на старте каждого прогона,
но реально дёргает API один раз на версию (флаг bot_commands_version в
bot_state); при неудаче флаг не ставится и попытка повторяется на
следующем прогоне. Ручной шаг в docs/bot-setup.md заменён описанием
автонастройки.

## Prevention

- Разовые настройки бота (команды, кнопка меню, описание через API)
  оформлять как идемпотентные ensure_*-функции с версионным флагом в
  bot_state, а не как ручные шаги в документации.
- При изменении списка команд не забывать поднять BOT_COMMANDS_VERSION —
  иначе перерегистрация не произойдёт (покрыто тестом
  test_new_version_reregisters).
